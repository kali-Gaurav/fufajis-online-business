ISSUE
-----
10 orphaned Firestore collections with ZERO security rules
- delivery_tracking (completion data)
- delivery_routes (routing data)
- delivery_assignments (assignment data)
- delivery_otp (OTP codes - PLAINTEXT!)
- delivery_agents (rider location data)
- delivery_locations (GPS coordinates - LEAKED!)
- delivery_status (status history)
- delivery_history (event logs)
- delivery_notifications (notification data)
- delivery_preferences (user preferences)

SEVERITY: P0 CRITICAL (Live bug in production)
IMPACT: GPS data publicly accessible, OTP codes exposed, no RLS


SOLUTION
--------
Consolidate all 10 collections into single DELIVERY_TASKS collection
with proper Firestore Row-Level Security (RLS)

Rider sees ONLY their own deliveries
Customer sees ONLY their own delivery
Dispatcher sees branch-scoped deliveries
Admin sees all deliveries

FIRESTORE RULES SUMMARY
-----------------------

DELIVERY_TASKS (NEW - SECURED)
  Read:   Admin + Dispatcher (branch-scoped) + Rider (self) + Customer (self)
  Create: Admin + Dispatcher only
  Update: Admin + Dispatcher (branch-scoped) + Rider (self updates only)

Deprecated Collections (READ-ONLY)
  Read:   Allowed (for migration period)
  Write:  BLOCKED (consolidated into delivery_tasks)
