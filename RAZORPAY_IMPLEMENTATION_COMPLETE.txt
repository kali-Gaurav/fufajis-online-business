================================================================================
KEY FEATURES IMPLEMENTED
================================================================================

✓ Proper Credential Separation
  - key_secret for API calls (create orders, refunds)
  - webhook_secret for signature verification
  - Validation at service initialization

✓ Secure Payment Flow
  1. Client creates order → Backend generates Razorpay order
  2. Client opens checkout → Razorpay processes payment
  3. Client verifies payment → Backend checks signature (webhook_secret)
  4. Backend creates order → Atomic transaction (order+payment+ledger)
  5. Razorpay webhook arrives → Verified with webhook_secret
  6. Webhook processed → Idempotent update (no duplicates)

✓ Atomic Transactions
  - Order + Payment + Ledger created together
  - All-or-nothing consistency
  - Prevents partial failures

✓ Idempotency
  - Webhook events tracked in webhook_events collection
  - Duplicate events skipped automatically
  - Prevents double-charging

✓ Amount Validation
  - Webhook amount checked against order amount
  - ₹1 rounding tolerance
  - Mismatches logged to reconciliation_log

✓ Error Recovery
  - Comprehensive logging to payment_reconciliation_log
  - Admin reconciliation endpoint
  - Manual fix capability

✓ Wallet Integration
  - Refunds restore customer wallet balance
  - Atomic with refund processing
  - Ledger entry for accounting

DATA MODEL
================================================================================

Collections Created/Updated:

payments
├─ paymentId, orderId, customerId
├─ amount, currency, method
├─ status (pending/captured/failed/refunded)
├─ verified, verifiedAt
├─ source (client/webhook)
├─ refundId, refundAmount, refundedAt
└─ createdAt, updatedAt

payment_ledger
├─ orderId, paymentId, customerId
├─ type (credit/debit)
├─ amount, method, reason
├─ status, timestamp
└─ refundId (if applicable)

webhook_events (Idempotency)
├─ razorpay_{eventId}
├─ eventId, orderId, paymentId
├─ type, amount
└─ processedAt, error

payment_reconciliation_log (Audit)
├─ action, paymentId, orderId
├─ event, error
├─ webhookAmount, orderAmount
└─ timestamp, signature
