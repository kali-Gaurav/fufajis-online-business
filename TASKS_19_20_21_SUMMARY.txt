================================================================================
ARCHITECTURE DIAGRAM
================================================================================

Payment Flow:

  Razorpay
     |
     | HTTPS POST (webhook)
     v
  Edge Function: razorpay-webhook-dual-write
     |
     +---> Verify Signature (HMAC-SHA256)
     |
     +---> PostgreSQL: Create payment_transactions (source of truth)
     |
     +---> Firebase Bridge: Sync to Firestore (real-time app)
     |
     +---> PostgreSQL: Update order status
     |
     +---> PostgreSQL: Deduct inventory
     |
     +---> PostgreSQL: Trigger push notification
     |
     v
  200 OK → Razorpay

Data Flow (Dual-Write):

  PostgreSQL (Source of Truth)
     ^
     | Read
     |
  Mobile App ←--- Firestore (Real-time Cache)
     |              ^
     | Write         |
     +------------→ +--- Synced by Edge Function

Storage Flow:

  File Upload (Frontend/Mobile)
     |
     v
  Supabase Storage Bucket (e.g., product-images)
     |
     | RLS Policy Check (auth.uid() + folder check)
     |
     +---> Storage_references table (track metadata)
     |
     v
  Signed URL (for private files)
     |
     v
  Cache in PostgreSQL (avoid repeated generation)

SUPPORT & TROUBLESHOOTING
================================================================================

Common Issues:

1. Firebase Admin SDK initialization fails
   Solution: Verify FIREBASE_SERVICE_ACCOUNT secret is set correctly
             Re-download service account JSON from Firebase Console

2. Storage bucket already exists error
   Solution: Normal - migrations are idempotent
             Buckets created by previous deployment will be skipped

3. Webhook signature verification fails
   Solution: Verify RAZORPAY_WEBHOOK_SECRET matches Razorpay dashboard
             Redeploy webhook function after updating secret

4. Firestore sync fails (non-fatal)
   Solution: This is expected (PostgreSQL is source of truth)
             Firebase sync is async and non-blocking
             Check Firebase credentials if persistent failure
