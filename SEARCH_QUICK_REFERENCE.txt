┌─ THREE SEARCH STRATEGIES ────────────────────────────────────────────────┐
│                                                                             │
│  1. BARCODE LOOKUP (Fastest - <20ms)                                       │
│     Direct indexed Firestore query                                         │
│     Perfect for: Barcode scanner integration                              │
│     Example: await searchService.searchByBarcode(shopId, "8901234567890") │
│                                                                             │
│  2. KEYWORD SEARCH (Fast - <50ms)                                          │
│     Exact match on searchKeywords field                                    │
│     Perfect for: Product name search                                       │
│     Example: await searchService.searchProducts(shopId, "rice")            │
│                                                                             │
│  3. FUZZY MATCH (Smart - <100ms)                                           │
│     Trigram-based matching handles typos                                   │
│     Perfect for: Typo tolerance (e.g., "ryce" → finds "rice")             │
│     Auto-triggered when keyword match returns nothing                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

┌─ COMMON QUESTIONS ───────────────────────────────────────────────────────┐
│                                                                             │
│  Q: How do I add this to my POS system?                                   │
│  A: Import ProductSearchScreen and push it when user needs product search.│
│     See lib/examples/search_integration_example.dart for code.             │
│                                                                             │
│  Q: Do I need to reindex products?                                         │
│  A: Yes, once. Call indexProductsForSearch(shopId) to populate search     │
│     fields. Takes ~30 seconds for 5000 products.                           │
│                                                                             │
│  Q: What if my database is offline?                                        │
│  A: ProductSearchService handles offline gracefully with error logging.   │
│     User will see "Search error" message.                                  │
│                                                                             │
│  Q: Can I customize the search UI?                                         │
│  A: Yes! ProductSearchScreen is fully customizable Flutter code.          │
│     Colors, spacing, text all configurable.                               │
│                                                                             │
│  Q: What about voice search?                                              │
│  A: Voice search coming in Week 3. This creates the foundation.           │
│                                                                             │
│  Q: How do I monitor performance?                                          │
│  A: Use Firebase Console Firestore → Performance tab. Search metrics      │
│     are logged in LoggingService for debugging.                           │
│                                                                             │
│  For more Q&A, see SEARCH_PERFORMANCE_GUIDE.md Troubleshooting section    │
