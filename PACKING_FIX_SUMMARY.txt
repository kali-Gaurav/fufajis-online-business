THE PROBLEM
───────────────────────────────────────────────────────────────────────────────

Delivery service could not find packed orders because two independent packing
workflows wrote to different Firestore collections with different status values:

  PackingService v1:
    Collection: fulfillment_tasks
    Statuses: new, assigned, picking, quality_check, verified, completed, rejected

  PackingService v2:
    Collection: fulfillment_tasks_v2
    Statuses: NEW, IN_PROGRESS, QUALITY_CHECK, COMPLETED, REJECTED

  DeliveryWorkflowService queried:
    WHERE order.status = 'packed' IN fulfillment_tasks
    → Orders in v2 collection invisible!

IMPACT: Orders disappear after packing (can't be delivered)


═══════════════════════════════════════════════════════════════════════════════

THE SOLUTION
───────────────────────────────────────────────────────────────────────────────

1. SINGLE COLLECTION
   Consolidate all packing into: fulfillment_tasks (DELETE v2 collection)

2. SINGLE STATE MACHINE
   Unified status flow: new → assigned → picking → quality_check → verified → completed
   Status mapping: NEW→new, IN_PROGRESS→assigned, COMPLETED→verified, etc.

3. MIGRATION SERVICE
   Safe, idempotent migration from v2→v1 with validation

4. DEPRECATION
   Mark all v2 methods as @Deprecated to prevent accidental usage

UNIFIED PACKING WORKFLOW
───────────────────────────────────────────────────────────────────────────────

PackingWorkflowService (SINGLE entry point):

  1. CREATE TASK
     createFulfillmentTask(orderId, shopId, branchId, items)
     → status: 'new'

  2. ASSIGN EMPLOYEE
     assignToEmployee(taskId, employeeId, employeeName)
     → status: 'new' → 'assigned'

  3. PICK ITEMS
     markItemPicked(taskId, itemId, quantity, batchNumber, expiryDate)
     → Auto-transition: 'assigned' → 'picking'
     → All items picked: 'picking' → 'quality_check'

  4. REQUEST QC
     requestQualityCheck(taskId)
     → status: 'picking' → 'quality_check'

  5. VERIFY ITEMS (Pass QC)
     verifyItems(taskId, verifiedBy, notes)
     → status: 'quality_check' → 'verified'
     → Triggers: Stock deduction, order update

  6. REJECT PACKING (Fail QC)
     rejectPacking(taskId, rejectionReason, rejectedBy)
     → status: 'quality_check' → 'rejected'
     → Resets: pickedItems, verifiedItems
     → Reassign: 'rejected' → 'assigned'

  7. COMPLETE PACKING
     markCompleted(taskId)
     → status: 'verified' → 'completed'
     → Updates: order.status = 'packed'
     → Creates: delivery task
