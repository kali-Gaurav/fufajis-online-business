TECHNICAL SPECIFICATIONS:

Database Schema:
- Table: offline_orders (9 columns)
- Fields: id, order_json, status, retry_count, last_retry_at,
          firestore_id, conflict_resolution_data, created_at, updated_at, synced_at
- Index on status for fast queries

Network Strategy:
- Auto-sync on: network reconnect + 5-minute timer
- Retry: Exponential backoff (1s, 2s, 4s)
- Max retries: 3 per order
- Sync strategy: One order at a time

Memory Efficiency:
- Metadata cache: ~100 bytes per order
- Full load on demand: ~50KB per order
- Auto-cleanup: Orders > 7 days old

Conflict Resolution:
- Strategy: Server-wins merge
- Metadata: Stored for audit trail
- Data loss: Zero (all data preserved)
